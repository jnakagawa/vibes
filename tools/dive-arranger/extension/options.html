Dive Arranger

The arranger first tries to reuse a MotherDuck token it can find in the app.motherduck.com tab's own storage. If it can't find one, it falls back to the token below.

Stored only in chrome.storage.local on this machine — never synced, never sent anywhere except api.motherduck.com by the wasm client. Short-lived tokens work and are the safest choice. The token needs read/write scope to save layouts.
